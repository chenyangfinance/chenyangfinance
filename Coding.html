    2. kmatch: the best matching tool in Stata

    kmatch is the most comprehensive tool in Stata to do matching. It is capable to suit a wide range of matching issues, including multivariate-distance and propensity-score matching (PSM), entropy balancing, inverse probability weighting, (coarsened) exact matching, and regression adjustment. The most useful function of kmatch is EXACT MATCHING !!! e.g., when you do PSM, you can use some variables (firm size, leverage, ROA, etc.) to calculate the score but require exact matching of some other variables (probably industry, year, state, etc.) at the same time.

    Easy to install with: 

    ssc install kmatch 

help kmatch

    3. ffind: convert SIC industry codes to Fama-French 30/48/49 codes

    Written by Judson Caskey, with UCLA,   ffind is an esay-to-use command to convert SIC codes to Fama-French industry codes. You will need to install from the website. 

    net from https://sites.google.com/site/judsoncaskey/data
net install utilities.pkg

    To use it, 

    ffind sic, newvar(ff48) type(48)

* This command converts SIC codes (variable name: sic) to Fama-French 48 codes (variable name: ff48)
* type() is the option to input the type of Fama-French codes, i.e., 30, 48, or 49.

    See here for an equivalent SAS macro: http://faculty-gsb.stanford.edu/dehaan/SAS.html
